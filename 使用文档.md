## 系统概述

本系统实现10台以内安卓设备并发自动化操作小红书，完成发现页目标用户帖子的自动识别、点赞、收藏、截图等操作。内置AI Agent引擎自主处理执行过程中的异常情况。

### 核心能力

- **多设备并发**: 支持最多10台安卓设备同时运行，单进程单设备隔离
- **智能内容识别**: 自动区分图文帖和视频帖，采用不同浏览策略
- **目标用户匹配**: 通过OCR识别发现页用户昵称，支持精确/模糊匹配
- **行为多样性**: 每个设备拥有独特的行为画像，模拟不同用户习惯
- **AI异常处理**: 基于LLM的异常决策引擎，自动处理卡点情况
- **账号保护**: 养号期策略+每日操作限制，降低风控风险
- **监控面板**: Web界面实时监控设备状态、任务进度、异常记录

## 环境搭建

### 系统要求

- **Python**: 3.10+
- **操作系统**: Windows 10/11, macOS, Linux
- **设备**: Android手机（USB或WiFi连接）
- **ADB**: AirTest框架自带，无需单独安装

### 安装步骤

#### 1. 安装Python依赖

```bash
pip install -r requirements.txt
```

核心依赖说明：
- `airtest>=1.3.0` - 设备控制和截图
- `poco>=1.12.0` - UI控件树识别
- `flask>=3.0.0` - 监控面板后端
- `openai>=1.0.0` - LLM调用（豆包模型）
- `opencv-python` - 图像模板匹配
- `easyocr` - OCR文字识别

#### 2. 连接安卓设备

**USB连接（推荐）**:
1. 手机开启开发者模式和USB调试
2. USB连接电脑
3. 验证连接：`adb devices`

**WiFi连接**:
1. 先用USB连接一次
2. 执行 `adb tcpip 5555`
3. 拔掉USB，执行 `adb connect <手机IP>:5555`

#### 3. 验证设备连接

```bash
adb devices
# 应显示已连接的设备列表
```

#### 4. 验证AirTest/Poco对小红书的UI控件识别

运行测试脚本验证设备连接和小红书控件识别是否正常。

## 快速开始

### 第一步：配置系统参数

编辑 `configs/system.yaml`，至少配置以下参数：

```yaml
system:
  max_devices: 10                  # 最大设备数
  operations_per_minute: 8         # 每分钟最多操作次数
  daily_limit_per_user: 3          # 每个目标用户每日最多操作次数

llm:
  api_key: "your-api-key"          # 火山引擎API密钥
  base_url: "https://ark.cn-beijing.volces.com/api/coding/v1"
  model: "Doubao-Seed-2.0-pro"
```

### 第二步：配置账号

编辑 `configs/accounts.yaml`：

```yaml
accounts:
  - account: "手机号"
    password: "密码"
    device: null    # null表示自动分配
    status: "active"
```

### 第三步：配置目标用户

编辑 `configs/targets.yaml`：

```yaml
target_users:
  - xiaohongshu_id: "用户ID"
    nickname: "用户昵称"
    match_mode: "exact"   # exact=精确匹配, fuzzy=模糊匹配
```

### 第四步：初始化系统

```bash
python main.py init
```

这会初始化SQLite数据库、创建必要的表结构和索引。

### 第五步：启动运行

```bash
# 启动完整系统（任务调度+监控面板）
python main.py all

# 或单独启动单设备测试
python main.py device

# 或单独启动监控面板
python main.py dashboard
```

监控面板访问地址：`http://localhost:8080`

## 配置说明

### system.yaml - 系统配置

#### 设备设置
| 参数 | 说明 | 默认值 |
|------|------|--------|
| max_devices | 最大设备数 | 10 |
| heartbeat_interval | 心跳间隔（秒） | 5 |
| offline_threshold | 离线判定阈值（秒） | 30 |

#### 设备黑白名单
```yaml
device_whitelist: ["设备ID1", "设备ID2"]  # 仅允许列表中的设备
device_blacklist: ["设备ID3"]            # 禁止列表中的设备
```

#### 操作限制
| 参数 | 说明 | 默认值 |
|------|------|--------|
| operations_per_minute | 每分钟最多操作次数 | 8 |
| daily_limit_per_user | 每个目标用户每日最多操作次数 | 3 |
| min_interval_between_operations | 同一用户操作最小间隔（秒） | 300（5分钟） |

#### 反爬设置
| 参数 | 说明 | 默认值 |
|------|------|--------|
| new_account_ramp_days | 新账号养号天数 | 7 |
| new_account_daily_limit | 新账号每日最多操作次数 | 15 |
| human_like_delay_min | 点击前最小延迟（毫秒） | 500 |
| human_like_delay_max | 点击前最大延迟（毫秒） | 2000 |
| human_swipe_speed_min | 滑动最小时长（毫秒） | 200 |
| human_swipe_speed_max | 滑动最大时长（毫秒） | 800 |
| human_browse_time_min | 浏览最小时间（毫秒） | 1000 |
| human_browse_time_max | 浏览最大时间（毫秒） | 5000 |

#### 账号轮换
```yaml
account_rotation:
  enabled: false        # 是否启用账号轮换
  rotation_period: 3600 # 轮换周期（秒）
```

#### AI Agent设置
| 参数 | 说明 | 默认值 |
|------|------|--------|
| enabled | 是否启用AI Agent | true |
| max_llm_concurrent | LLM最大并发数 | 3 |
| llm_timeout | LLM超时时间（秒） | 30 |
| case_similarity_threshold | 案例匹配阈值 | 0.85 |

#### 监控面板
```yaml
dashboard:
  host: "0.0.0.0"  # 监听地址
  port: 8080       # 监听端口
```

## 功能模块说明

### 设备管理（device-manager）

- 自动扫描连接的USB/WiFi设备
- 心跳监控和断线重连（最多重试3次）
- 设备行为画像：每个设备基于ID生成独特的行为参数，模拟不同用户习惯
  - 滑动速度系数、滑动轨迹偏移、点击延迟、浏览速度、内容偏好

### 账号管理（account-manager）

- 自动登录小红书（填充账号密码）
- 验证码人工提醒（桌面通知+声音告警）
- 账号轮换（按配置的周期自动切换）
- 登录状态校验与自动重新登录

### 真人操作模拟（human-simulation）

- 随机滑动距离（300-800px）
- 随机滑动时长（200-800ms）
- S型滑动轨迹算法（带随机偏移）
- 随机点击延迟（500-2000ms）
- 随机页面加载等待（1000-5000ms）

### 业务流程（xiaohongshu-workflow）

**完整操作流程**:
1. 启动小红书APP → 进入发现页
2. 发现页截图（匹配到目标用户时保存）
3. 识别目标用户帖子（OCR昵称匹配）
4. 进入帖子详情
5. 自动识别内容类型（图文/视频）
6. 浏览内容：
   - 图文：滑动浏览至最后一页
   - 视频：拖动进度条至2/3位置
7. 点赞和收藏（行为随机化）
8. 操作完成截图（保存到用户目录）
9. 返回发现页继续

**行为多样性随机化**:
- 80%概率点赞+收藏，15%仅点赞，5%仅收藏
- 随机操作顺序（先点赞或先收藏）
- 30%概率先在空白区域点击一次
- 40%概率操作后随机滑动（模拟看评论）
- 操作间随机延迟

### AI Agent（ai-agent-engine）

- 异常感知：屏幕截图、控件树、日志
- 案例库检索：精确匹配+相似度检索
- LLM决策：豆包模型Vision分析
- 行动校验：坐标合法性、参数完整性
- 自学习：成功案例入库、失败案例标记

## 账号保护策略

系统内置账号保护机制，防止新账号因操作频繁被风控：

### 养号期策略

- **新账号养号期**: 7天
- **新账号每日限制**: 15次操作
- **成熟账号每日限制**: 50-100次操作（可配置）

### 保护机制工作流程

1. 每次循环开始前检查账号状态
2. 判断账号是否在养号期内（创建时间<7天）
3. 如果是新账号且今日操作已达上限，暂停运行并休眠
4. 每次成功操作后增加今日操作计数
5. 每日零点自动重置操作计数

### 查看账号状态

通过监控面板或数据库查询：
- 账号创建时间
- 已使用天数
- 今日已操作次数

## 反爬规避机制

### 操作频率控制

- 每分钟最多8次操作（平均每7.5秒一次）
- 每个目标用户每天最多3次操作
- 同一用户操作间隔至少5分钟

### 行为多样性

- 每个设备有独特的行为画像
- 随机化滑动距离、时长、轨迹
- 随机化点击延迟和浏览时间
- 随机化操作模式（点赞/收藏组合）

### 账号保护

- 新账号7天养号期
- 每日操作次数限制
- 账号轮换支持

## AI Agent 异常处理

### 何时触发

当系统运行过程中出现异常（找不到元素、页面卡住、APP崩溃等），自动触发AI Agent。

### 工作流程

1. **异常感知**: 采集当前截图、控件树、最近10条日志
2. **案例库检索**: 查找是否有相同场景的历史成功案例
3. **LLM决策**: 无可用案例时，调用豆包大模型分析截图和控件树
4. **行动执行**: 解析LLM返回的JSON指令，执行点击/滑动等操作
5. **自学习**: 将成功案例存入案例库，失败案例标记并减少使用优先级

### 人工介入

当AI Agent无法自动处理时，会触发人工告警：
- 桌面弹出通知
- 系统暂停等待人工处理
- 人工处理完成后自动恢复

## 常见问题排查

### 设备连接问题

**Q: adb devices 看不到设备**
- 检查USB线是否连接正常
- 确认手机已开启USB调试
- 尝试重新插拔USB

**Q: 设备频繁断线**
- 检查USB集线器供电是否稳定
- 尝试更换USB线
- 系统会自动重连3次

### 登录问题

**Q: 自动登录失败**
- 确认账号密码正确
- 检查网络是否正常
- 如出现验证码，系统会暂停并等待人工输入

**Q: 登录状态失效频繁**
- 检查APP是否需要定期验证
- 考虑增加登录状态校验间隔

### 业务流程问题

**Q: 找不到目标用户**
- 确认targets.yaml中昵称与实际一致
- 检查OCR识别是否正常工作
- 尝试调整匹配模式（exact/fuzzy）

**Q: 点赞/收藏未执行**
- 检查模板图片是否存在于templates/目录
- 确认小红书APP版本未大幅更新UI

### AI Agent问题

**Q: LLM调用超时**
- 检查网络连接是否正常
- 确认API key是否有效
- 系统有30秒超时降级机制

**Q: 案例库匹配不到**
- 正常情况，新场景无历史案例时会调用LLM
- 随着运行时间增长，案例库会逐渐丰富
